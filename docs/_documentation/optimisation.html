Optimization with polynomial surrogates¶

Perform unconstrained or constrained optimization.

class equadratures.optimization.Optimization(method='trust-constr')[source]¶

This class performs unconstrained or constrained optimization of Poly objects using scipy.optimize.minimize.

Parameters

method (optional) (string) – A string specifying the method that will be used for optimization. All of the available choices come from scipy.optimize.minimize (see <https://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.minimize.html> for a list of methods and further information). In the case of general constrained optimization, the options are ‘COBYLA’, ‘SLSQP’, and ‘trust-constr’. The default is ‘trust-constr’.

addLinearEqCon(A, b)[source]¶

Adds linear equality constraints A x = b to the optimization routine. Only ‘trust-constr’ and ‘SLSQP’ methods can handle equality constraints.

Parameters

  • A (ndarray) – M-by-n matrix that contains coefficients of the linear equality constraints

  • b (ndarray) – M-by-1 matrix that specifies right hand side of the linear equality constraints.

addLinearIneqCon(A, b_l, b_u)[source]¶

Adds linear inequality constraints b_l <= A x <= b_u to the optimization problem. Only ‘trust-constr’, ‘COBYLA, and ‘SLSQP’ methods can handle general constraints.

Parameters

  • A (ndarray) – M-by-n matrix that contains coefficients of the linear inequality constraints

  • b_l (ndarray) – M-by-1 matrix that specifies lower bounds of the linear inequality constraints. If there is no lower bound, set b_l = -np.inf * np.ones(M).

  • b_u (ndarray) – M-by-1 matrix that specifies upper bounds of the linear inequality constraints. If there is no upper bound, set b_u = np.inf * np.ones(M).

addNonLinearEqCon(value, Poly=None, subspace=None, Function=None, jacFunction=None, hessFunction=None)[source]¶

Adds nonlinear inequality constraints g(x) = value to the optimization routine. Only ‘trust-constr’ and ‘SLSQP’ methods can handle equality constraints. If Poly object is provided, gradients and Hessians will be computed automatically. If ‘subspace’ is provided, this method adds the constraint g(M^T x) = value, where M is a tall matrix specified using the subspace argument. If a lambda function is provided with ‘function’ argument, the user may provide ‘jacFunction’ for gradients and ‘hessFunction’ for Hessians; otherwise, a 2-point differentiation rule will be used to approximate these.

Parameters

  • value (float) – a float value specifying what nonlinear function must be equal to

  • Poly (optional) (Poly) – an instance of the Poly class

  • subspace (optional) (ndarray) – n-by-d matrix specifying the tall subspace matrix

  • Function (optional) (function) – a lambda function of constraint

  • jacFunction (optional) (function) – a lambda function of the gradient of the constraint

  • hessFunction (optional) (function) – a lambda function of the Hessian of the constraint

addNonLinearIneqCon(bounds, Poly=None, subspace=None, Function=None, jacFunction=None, hessFunction=None)[source]¶

Adds nonlinear inequality constraints lb <= g(x) <= ub to the optimization problem, where lb, ub = bounds. Only ‘trust-constr’, ‘COBYLA, and ‘SLSQP’ methods can handle general constraints. If Poly object is provided using ‘poly’ argument, gradients and Hessians will be computed automatically. If ‘subspace’ is provided, adds the constraint lb <= g(M^T x) <= ub, where M is a tall matrix specified using the subspace argument. If a lambda function is provided with ‘function’ argument, the user may provide ‘jacFunction’ for gradients and ‘hessFunction’ for Hessians; otherwise, a 2-point differentiation rule will be used to approximate these.

Parameters

  • bounds (ndarray) – array with 2 entries specifying the lower and upper bounds of the inequality. If there is no lower bound, set bounds[0] = -np.inf. If there is no upper bound, set bounds[1] = np.inf.

  • Poly (optional) (Poly) – an instance of the Poly class

  • subspace (optional) (ndarray) – n-by-d matrix specifying a subspace matrix

  • Function (optional) (function) – a lambda function of constraint

  • jacFunction (optional) (function) – a lambda function of the gradient of the constraint

  • hessFunction (optional) (function) – a lambda function of the Hessian of the constraint

optimizePoly(x0)[source]¶

Performs optimization on a specified function, provided the objective has been added using ‘addObjective’ method and constraints have been added using the relevant method.

Parameters

Returns
