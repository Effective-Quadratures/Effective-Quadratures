Parameter¶

A parameter—i.e., the definition of a univariate variable \(x\)—is one the main building blocks in Effective Quadratures. It characterized by its distribution (and support) \(\rho(x)\).

Core class for setting the properties of a univariate parameter.

References:
  • Akil Narayan Paper on Induced Distributions Paper.
  • Walter Gautschi resources on orthogonal polynomials
  • Tiziano’s paper on adaptive polynomial expansions
class equadratures.parameter.Parameter(order, lower=None, upper=None, distribution=None, shape_parameter_A=None, shape_parameter_B=None, data=None, endpoints=None)[source]¶

This class defines a univariate parameter. Below are details of its constructor.

Parameters:
  • lower (double) – Lower bound for the parameter.
  • upper (double) – Upper bound for the parameter.
  • order (integer) – Order of the parameter.
  • param_type (string) – The type of distribution that characterizes the parameter. Options include: Chebyshev (arcsine), Gaussian, Truncated-Gaussian, Beta, Cauchy, Exponential, Uniform, Gamma, Weibull. If no string is provided, a Uniform distribution is assumed. If the user provides data, and would like to generate orthogonal polynomials (and quadrature rules) based on the data, they can set this option to be Custom.
  • shape_parameter_A (double) – Most of the aforementioned distributions are characterized by two shape parameters. For instance, in the case of a Gaussian (or TruncatedGaussian), this represents the mean. In the case of a Beta distribution this represents the alpha value. For a uniform distribution this input is not required.
  • shape_parameter_B (double) – This is the second shape parameter that characterizes the distribution selected. In the case of a Gaussian or TruncatedGaussian, this is the variance.
  • data – A numpy array with data values (x-y column format). Note this option is only invoked if the user uses the Custom param_type.
  • Endpoints – A boolean entry. If set to True, then the quadrature points and weights will have end-points.
